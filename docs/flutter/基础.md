# 基础

## 组件

### Container

> 是 Flutter 里最核心、最常用的 “多功能布局盒子”，相当于 Web 里的 div，能控制尺寸、边距、背景、边框、对齐、变形等，内部只容纳 1 个子组件

### Padding

> Padding 是 Flutter 里专门用来设置内边距的基础组件，作用就是：给子组件「加留白」，让组件和边框之间有间距。它非常纯粹：只负责内边距，只接收一个子组件，没有其他多余功能。

### Expanded

> Expanded 是 Flutter 中专门用在 Row / Column / Flex 内部的弹性布局组件，作用是：让子项在主轴方向上 “吃掉” 剩余空间，并可按比例分配

## 关键字

### const

> const 是 Dart 语言中的一个关键字，用于定义编译时常量。当你在 Flutter 中使用 const 修饰一个 Widget 时，表示这个 Widget 是不可变的，并且在编译时就已经确定了它的值。使用 const 可以提高性能，因为编译器可以在编译时进行优化，避免在运行时重新创建相同的 Widget 实例。

### final

> 在 Flutter（Dart 语言）里，final 是用来定义「只能赋值一次的常量变量」的关键字，核心作用就是：赋值后不可修改，是 Flutter 开发中最常用的关键字之一。

- #### 简单记

  > 不知道用啥 → 优先用 final
  > 确定是写死的编译时常量 → 用 const

### ClipRRect

> 用于将子组件裁剪为圆角矩形的核心组件，适用于图片、容器、卡片、头像等场景的圆角实现，尤其适合子组件自身不支持圆角或需要统一裁剪的情况

### AnimatedCrossFade

> 是 Flutter 内置的双组件交叉淡入淡出 + 尺寸自动过渡动画组件，用于在两个子组件间平滑切换，同时自动处理宽高变化，无需手动管理动画控制器

### UnconstrainedBox

> 是 Flutter 中打破父组件约束、让子组件 “自由” 布局的核心组件，它的作用是：解除父组件对子组件的大小限制，让子组件按照自身原始尺寸渲染，而不是被迫服从父级的约束。
> 简单说：父组件逼子组件必须多大，UnconstrainedBox 可以让子组件 “不听”，用自己的大小。

## 函数

### 命名参数与位置参数

```Dart
/// 命名参数
class ToolCategory {
  final String title;
  final String subtitle;
  final int count;
  final IconData icon;
  final List<String> tools;
  ToolCategory({
    required this.title,
    required this.subtitle,
    required this.count,
    required this.icon,
    required this.tools,
  });
}

/// 位置参数
class ToolCategory {
  final String title;
  final String subtitle;
  final int count;
  final IconData icon;
  final List<String> tools;
  ToolCategory( this.title, this.subtitle, this.count, this.icon, this.tools)
}

```
